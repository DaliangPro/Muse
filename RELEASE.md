# Muse 发布与签名说明

## 一、签名证书（"那支笔"）—— 最重要

Muse 用本机钥匙串里的 **`Muse Local`** 自签名证书签名（带私钥）。
macOS 把用户的麦克风/辅助功能授权**绑在这张证书上**——只要每次发布都用它签名，
用户授权一次后，无论发多少次更新都**不会失效**。

### 证书备份（防丢）
- 备份文件：`~/Documents/muse-signing-cert-backup.p12`
- 打开密码：`muse2026`
- **用途**：换电脑 / 钥匙串被清后，双击这个 `.p12` 输密码导入，即可恢复签名能力。
- ⚠️ **千万别丢、别公开**（含私钥）。建议另存一份到密码管理器 / 加密云盘。
- **丢失后果**：下次只能用新证书签名 → 所有老用户授权失效、需重新授权。

## 二、发布步骤

```bash
# 1. 编译 + 打包（自动复用 Muse Local 签名，不会退回 ad-hoc）
bash scripts/package-app.sh

# 2. 打成 .dmg
bash scripts/build-dmg.sh

# 3. 把 dist/ 下的 .dmg 传到 GitHub Release
```

每次都走这个流程 → 始终用同一张 Muse Local 证书 → 用户授权稳定保持。
**不要手动用 `codesign -s -`（ad-hoc）或重新生成证书**，否则授权会失效。

## 三、给用户的安装说明（贴进 GitHub Release / README）

> **首次打开**：本应用为免费自签名（未做 Apple 公证），首次打开 macOS 会提示
> "无法验证开发者"。解决办法：**右键点 Muse.app → 选「打开」→ 再点「打开」**，只需一次，
> 之后正常双击。（macOS 15+ 若仍被拦，到 系统设置 → 隐私与安全性 → 点「仍要打开」）
>
> **授权**：首次使用会依次请求**麦克风**、**辅助功能**权限，按提示授权即可。
> 其中辅助功能授权后需**重启一次 Muse** 才生效（macOS 规则）。

## 四、可选：消除"无法验证开发者"提示

如果想让用户双击直接打开（无警告），需要：
- **Apple Developer 账号**（$99/年）
- 用 **Developer ID Application** 证书签名 + **公证（notarize）**

这只影响"首次打开是否有警告"，**不影响授权稳定性**——授权稳不稳只取决于"是否每次用同一证书签名"。
