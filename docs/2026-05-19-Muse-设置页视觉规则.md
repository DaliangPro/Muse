# Muse 设置页视觉规则

## 核心方向

设置页不是营销页，也不是数据大屏。它应该是轻、稳、清楚的 macOS 工具界面：信息密度可以高，但视觉层级必须克制。

## 色彩

- 主背景使用温润浅灰，不使用纯冷灰。
- 大面积容器只承担分区，不承担装饰。
- 彩色只用于状态、资产类型、关键数据，不用于普通背景堆叠。
- 数据卡可以使用极浅色底，但不能加顶部色条或重边框。

## 字体

- 页面主体以 10、11、12、13 号为主。
- 只有关键数值允许使用 24 到 26 号。
- 标题不靠放大解决层级，优先靠位置、留白、字重解决。
- 同一行左右两侧文案必须保持字号和字重一致。

## 对齐

- 同一信息块内必须先定义唯一文本列，标题、正文、状态、输入框内容都按同一左边界对齐。
- 如果输入框或内容块有内边距，外部标题和操作区也必须使用相同横向内边距，禁止标题贴外框、正文缩进一截。
- 页面顶部工具栏、下方内容区、输入框标题、输入框正文、示例区标题与正文必须按同一文本列核对，不能只修局部。
- 右侧操作按钮按控件外边界对齐，必须和输入框、内容框、顶部右侧按钮的外边界一致；正文文字右边界按输入框内边距处理。
- 调整 UI 前先检查文字左边界；对齐优先级高于装饰、圆角和背景层级。

## 容器

- 一个信息块最多一层背景。
- 内部内容靠对齐、间距、轻微色差区分，避免多层矩形套矩形。
- 需要强分区时，用背景块；需要弱分区时，用留白；尽量少用线。

## 卡片

- 卡片圆角默认 12 到 16。
- 卡片之间靠 8 到 12 的间距建立节奏。
- 卡片内边距默认 14 到 16。
- 关键数据卡允许有轻色底和色彩数值；普通设置卡保持中性。
- 调整卡片、按钮、弹窗等容器尺寸时，必须同步检查内部 padding、子控件 frame、文字列、右侧操作边界、圆角和边缘内缩；禁止只改外层 frame 后默认认为内部会自动合理适配。
- 同一组 UI 的外边界、内容内边距、圆角和视觉边缘必须作为一个整体修改。除非明确只改外壳，否则尺寸改动默认包含内部内容的对齐与留白复核。

## 语料资产页

- 左侧是分类和候选列表，右侧是当前候选详情。
- 左侧列表不承载长解释，只保留类型、数量、候选标题、评级。
- 右侧详情优先展示：类型、时间、标题、标签、提炼内容、提取依据、操作。
- 原始输入默认折叠，不作为主信息展示。

## 提炼规则页

- 上半区：左侧规则类型，右侧 Prompt 与标准。
- 下半区：提炼策略。
- 全局规则和单项规则都可以恢复默认，但按钮文案必须明确作用范围。
- Prompt 是功能内容，不是说明文案；页面不放项目沟通式解释。

## 判断标准

看起来“干净”不等于好。最终判断以三点为准：

1. 用户第一眼知道当前页面在管什么。
2. 用户知道主要操作在哪里。
3. 页面有呼吸感，但没有无意义留白。
