# Muse 浅色面板配色规范

> 最后更新：2026-06-20 ·
> 适用范围：**设置面板**（不含 HUD 语音输入条）·
> 深色主题不受本规范约束、不得在浅色调整中连带改动

## 一、总方向

**「中性暖白 · 克制高级」。**

摒弃旧版的浅绿 / 暖米黄基调（毛玻璃会透出桌面色、随环境变色，且显旧、显廉价），统一为去黄、提亮、降饱和的中性暖白——干净、克制，接近 macOS 原生工具质感。

## 二、范围与边界（硬约束）

- 本规范只约束**设置面板**配色；**HUD 语音输入条不在范围**，不得连带改动。
- 只定义**浅色**；**深色色值在浅色调整中一律不动**。

## 三、核心色板（浅色 sRGB）

| 角色 | 值 | 说明 |
|---|---|---|
| 画布底（基层） | 0.923 / 0.919 / 0.911 | 内容区底色，统一下沉一档以衬托白卡浮起 |
| 白卡（浮起层） | 0.995 / 0.994 / 0.991 | 卡片、统计卡，靠明度差浮在画布之上 |
| 段切换轨道 | 0.879 / 0.875 / 0.867 | 比画布略深半档，自成凹槽区域 |
| 侧栏毛玻璃白罩 | 白 @ 55% alpha | 压住毛玻璃透出的桌面色 |
| 主文字 | 0.110 / 0.120 / 0.150 | 暖黑灰、非纯黑 |
| 强调色 | 琥珀（暂定） | 蓝（#3D6FD1）vs 琥珀尚未最终拍板 |

> 具体 token 见 `Muse/UI/DesignSystem.swift` 的 `TF.settings*` 系列。

## 四、设计原则

1. **干净、克制、扁平优先**；强调色点到为止——蓝一多、染色一重就廉价。
2. **不要突兀、偏硬的阴影**；但「浮起 / 图层感」可接受**柔到化开、只剩体积**的微投影（浮起体积的必要手段）。
3. **区分元素优先靠明暗 / 色彩本身分层，而非线条勾边**（描边方案已被明确否决）。
4. **深色色值在浅色工作中绝不连带改动。**

## 五、关键决策记录

- **段切换 / 开关面板**：取消内凹凹槽阴影、改纯平底；画布上的轨道改用比背景略深的实色区分（**方案 B**），不用描边、不用阴影。
- **概览页数据卡片的「图层感」**：**明暗分层**（画布下沉 + 白卡保持亮）打底，叠**一层柔化到几乎无形的微投影**补足浮起体积。先后否决了「描边」「仅加深染色」「仅明暗分层」等不达标方案——结论：在接近纯白的卡片上，要真正的浮起感，一点微投影不可省。
- **强调色**：当前保留旧版琥珀；是否改为克制蓝（#3D6FD1）待最终拍板。
