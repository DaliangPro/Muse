# Liquid Glass 研究结论

## 目的
这份文档只回答三件事：

1. 苹果官方所说的 `Liquid Glass` 到底是什么。
2. 我们前面的实现为什么会跑偏。
3. 接下来如果继续做，应该验证什么，不该再验证什么。

## 官方结论

### 1. Liquid Glass 不是“默认一直在流动的玻璃”
根据苹果官方 WWDC25 的描述，`Liquid Glass` 的核心不是持续播放动画，而是：

- 玻璃材质本身具备透镜感、折射感、分层感。
- 在交互、状态切换、背景内容经过、滚动边缘变化时，材质会做自适应响应。
- 静止时，整体应保持安静、轻、克制。

换句话说，苹果强调的是：

- 平时安静
- 事件发生时才活
- 活的是“光学响应”和“层级关系”，不是一层常驻特效

### 2. Liquid Glass 的“液态感”主要来自上下文响应
苹果官方材料里反复强调了几个来源：

- `lensing`：对背后内容的弯折和聚光
- `adaptivity`：根据背景亮暗、内容滚动、上下文变化去调整 tint、动态范围和阴影
- `morphing / flexing`：交互时的弹性和形变
- `grouping / merging`：多个 glass 元素在同一功能层上形成一致的整体

这意味着它的“液态感”不是一个独立动画，而是来自：

- 玻璃对背景内容的被动响应
- 玻璃对交互和结构变化的主动响应

### 3. 官方 API 给的是“玻璃底座”，不是完整成品
苹果提供了这些公开能力：

- `glassEffect`
- `GlassEffectContainer`
- `NSGlassEffectView`
- `NSGlassEffectContainerView`

这些 API 能提供：

- 玻璃材质
- 分层
- 基本的适应性
- 一部分合并和系统控件行为

但它们不会自动把任意自定义 HUD 变成“和系统控件一样的 Liquid Glass 成品”。

## 我们前面为什么跑偏

### 错误 1：把“液态感”做成了持续流动动画
前面多轮原型都犯了同一个错误：

- 让玻璃里一直有光在流
- 让染色层一直漂
- 让材质层自己不断表演

这和苹果逻辑相反。

苹果更像是：

- 默认静止
- 背景内容经过时局部产生折射、牵引
- 事件过去后回到静止

### 错误 2：把“玻璃里的光”当成了目标，而不是“玻璃对背景的响应”
我们前面做了很多：

- 内部染光
- 高光漂移
- 蓝绿流体层

这些都容易变成“玻璃上多了一层效果”，而不是“玻璃本体在响应背后内容”。

真正更接近苹果的目标应该是：

- 后面有东西经过
- 玻璃局部把那东西拧动、弯折、提亮、压暗
- 你先感知到“玻璃在响应”，而不是“有个特效层在动”

### 错误 3：把验证目标搞错了
我们之前一直在验证：

- 玻璃自己动起来好不好看

但真正该验证的是：

- 玻璃在静止时够不够安静
- 当高对比内容从后面经过时，局部响应够不够像苹果

## 对 Muse HUD 的具体约束

### 1. 这个 HUD 的场景先天不等于系统工具栏
Muse 的 HUD 是：

- 自定义悬浮 `NSPanel`
- 自定义内容结构
- 自定义波形和文本逻辑

它不是苹果系统工具栏、标签栏、导航栏那种原生宿主。

所以不能假设：

- 只要接入 `NSGlassEffectView`
- 就会自动出现和系统控件同级别的液态行为

### 2. 这个 HUD 不适合“满壳流动”
因为它体积小、功能单一，而且必须保证：

- 文本可读
- 首帧稳定
- 录音语义清晰

如果整块 HUD 一直流动，会立刻变假，也会干扰语义。

### 3. 真正值得验证的是“左侧局部被动响应”
对这个 HUD 来说，最合理的目标不是“整条玻璃壳变成液体”，而是：

- 左侧波形附近作为事件中心
- 当背景高对比内容从其后经过时
- 玻璃左半边局部出现很短暂的折射、牵引、透镜感
- 其它区域仍然安静

## 接下来正确的原型验证方式

如果继续做，原型必须满足下面这套验证逻辑。

### 原型原则

1. 默认静止
2. 不做持续流动
3. 背景里必须有高对比对象经过
4. 玻璃只对“经过事件”做局部响应
5. 响应结束后恢复静止

### 原型应该验证的东西

- 背后对象经过时，局部是不是被明显拧动了
- 这种拧动是不是发生在波形附近
- 这种拧动是不是像透镜/折射，而不是像一层蓝色特效
- 事件结束后，玻璃能不能回到安静状态

### 原型不该再验证的东西

- 整块玻璃一直流动好不好看
- 玻璃里漂一层蓝绿云雾好不好看
- 高光条一直扫来扫去好不好看

这些方向已经被事实证明会把效果做假。

## 现在的结论

### 已确认

- 纯 `NSGlassEffectView` 底座不够，无法自动得到强烈液态感。
- “持续流动”的思路是错的。
- “玻璃对背景的被动响应”才是正确方向。

### 还未验证

- 在 Muse 这个 HUD 场景里，是否能通过“局部被动折射响应”做到主观上足够像苹果。

这件事现在还没有被证明，也还没有被证伪。

## 下一步建议

只做一个新的隔离原型：

- HUD 默认静止
- 后面放一个明确的高对比对象缓慢经过
- 只让左侧局部玻璃对它产生短暂折射响应
- 不做任何常驻流动特效

只有这个原型成立，才有资格继续往正式版迁。
